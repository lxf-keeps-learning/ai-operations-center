# 数据库初始化说明

本文档说明 `AIOperationsCenter` Sprint0 本地 MySQL 初始化方式。当前本地开发不依赖 Docker，数据库通过本机 MySQL/Navicat 创建；Docker Compose 仅保留为后续标准部署方式。

## 1. Navicat 创建 ioc_ai 数据库

在 Navicat 中连接本机 MySQL 后：

1. 右键连接，选择“新建数据库”。
2. 数据库名填写 `ioc_ai`。
3. 字符集选择 `utf8mb4`。
4. 排序规则选择 `utf8mb4_unicode_ci`。
5. 确认创建。

可以在 Navicat 查询窗口执行：

```sql
CREATE DATABASE IF NOT EXISTS ioc_ai
  DEFAULT CHARACTER SET utf8mb4
  DEFAULT COLLATE utf8mb4_unicode_ci;

CREATE USER IF NOT EXISTS 'ioc_user'@'localhost' IDENTIFIED BY 'ioc_password';
GRANT SELECT, INSERT, UPDATE, DELETE, CREATE, ALTER, DROP, INDEX
  ON ioc_ai.* TO 'ioc_user'@'localhost';
FLUSH PRIVILEGES;
```

## 2. 配置 DATABASE_URL

后端只从 `DATABASE_URL` 读取数据库连接。

```bash
cd /Users/lxf/Documents/xinao/AIAgent/AIOperationsCenter/backend
cp .env.example .env
```

确认 `backend/.env` 中包含：

```text
DATABASE_URL=mysql+pymysql://ioc_user:ioc_password@localhost:3306/ioc_ai
```

如果你的 MySQL 端口、用户名或密码不同，只需要修改 `DATABASE_URL`。

MySQL 是当前 Sprint0 必须跑通的主数据库，Demo CRUD 接口会通过 SQLAlchemy 读写 `system_items` 表。

## 3. Redis 本地开发说明

当前本地开发模式 Redis 可不启用。确认 `backend/.env` 中保留：

```text
REDIS_ENABLED=false
REDIS_URL=redis://localhost:6379/0
```

当 `REDIS_ENABLED=false` 时，后端不会在启动时连接 Redis，Redis 未启动也不会影响 MySQL、接口文档或 Demo CRUD。后续标准部署模式可将 `REDIS_ENABLED` 改为 `true` 并配置实际 `REDIS_URL`。

Redis 是后续缓存、会话、任务状态、限流能力的预留组件；Sprint0 只保留配置和健康检查，不引入复杂缓存业务。

健康检查接口：

```bash
curl http://localhost:8000/api/cache/ping
```

默认返回 `Redis disabled`。启用 Redis 后，该接口会尝试连接并返回 `pong`；连接失败时会返回清晰错误信息，不影响其他接口。

## 6. 启动后端

默认端口：

```bash
fastapi dev app/main.py --host 127.0.0.1 --port 8000
```

如果 `8000` 被占用，可使用：

```bash
uvicorn app.main:app --host 127.0.0.1 --port 8001 --reload
```

## 8. 验证 Demo CRUD

创建 Item：

```bash
curl -X POST http://localhost:8000/api/items \
  -H "Content-Type: application/json" \
  -d '{"name":"Sprint0 Demo","description":"MySQL CRUD demo","is_active":true}'
```

查询列表：

```bash
curl http://localhost:8000/api/items
```

详情、更新、删除：

```bash
curl http://localhost:8000/api/items/1

curl -X PUT http://localhost:8000/api/items/1 \
  -H "Content-Type: application/json" \
  -d '{"description":"updated by Sprint0"}'

curl -X DELETE http://localhost:8000/api/items/1
```

## 9. Docker Compose 说明

项目根目录 `docker-compose.yml` 保留为后续标准部署方式，当前本地启动不依赖 Docker。后续需要容器化 MySQL 时可执行：

```bash
cd /Users/lxf/Documents/xinao/AIAgent/AIOperationsCenter
docker compose up -d mysql
```

Docker 模式下真实数据保存在 Docker volume `mysql_data` 中，不提交到 Git。
