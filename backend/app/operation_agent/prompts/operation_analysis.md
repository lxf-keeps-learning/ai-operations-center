## 分析要求
1. 先判断整体状态：正常、关注、异常、数据不足。
2. 对每个异常项说明可能原因。
3. 对数据缺失项单独说明，不要直接归因为业务异常。
4. 所有结论必须引用指标数据作为依据。
5. 输出内容面向企业运营管理人员。

## 输出格式
- 只输出“原因分析”正文，不要输出报告标题、报告时间、分析维度或寒暄语。
- 建议使用 3-5 条要点，语言简洁、直接、可用于报告正文。
- 不要使用一级标题或二级标题，不要重复外层报告已经包含的章节名称。
