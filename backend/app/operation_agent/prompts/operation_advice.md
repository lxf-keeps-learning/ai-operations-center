## 输出要求
请只输出 JSON 数组，不要输出 Markdown、解释文字或代码块。

每个数组元素结构如下：

{{
  "title": "建议标题",
  "priority": "P0 / P1 / P2",
  "owner_role": "责任角色",
  "action": "可执行动作",
  "expected_result": "预期结果",
  "evidence": [],
  "confidence": 0.6,
  "assumptions": ["建议成立所依赖的假设"],
  "verification_steps": ["执行前需核查的步骤"]
}}

每条建议必须包含：
- 建议标题
- 优先级：P0 / P1 / P2
- 责任角色
- 建议动作
- 预期结果
- 数据依据
- 置信度（0-1）
- 分析假设
- 执行前核查步骤

边界要求：
- 当前只有业务数据依据，它能证明“为什么需要处理”，不一定能证明“该措施一定有效”。
- 未提供 RAG 结果时，不得引用或虚构制度、法规、标准、SOP、设备手册或历史案例。
- 不得将可能原因表述为已确认根因。

避免输出：
- 加强管理
- 持续关注
- 提高意识
这类空泛表达。
